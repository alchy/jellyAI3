14 · Architektura karet

Karty jsou jediný zdroj chování systému: deklarativní definice, rodiny s dimenzemi, kompilace do otázkového grafu a rozvinutí do konkrétních vzorů.

14.1 Anatomie karty

Karta = jeden JSON soubor v jellyai/iris/patterns/cs/ = jeden vzor chování. Čtyři části:

KlíčVýznamPříklady obsahu
triggerkdy sedímevent (přesná shoda s událostí automatu), pattern (vzor tříd lexeru), assurance_below, min_candidates, requires/forbids (rysy), priority
dialogco říkámšablona věty: „Mohu doplnit: {roles}.“
actionco dělámwarm_candidates, await, query (překlad vzoru na dotaz)
teachjak se vysvětlujivýklad vzoru pro člověka (a pro tuto dokumentaci)

Past 3/13: uvnitř teach řetězců nikdy ASCII uvozovky " — vždy „české“. Rozbitá karta = desítky padlých testů.

14.2 Dvě podoby triggeru: rysy × vzor

Rysové karty (starší) žádají rysy tahu: requires/forbids + prahy. Vzorové karty (#46) nesou pattern — regulární sekvenci tříd lexeru; vybírají se matchem sekvence, ne rysy. Obě rodiny sdílejí jednu prioritu decku — e-mailový atribut (priorita 12) tak správně přebije kartu krátkých sloves (8) na větě „Karel má email…“.

Vzor smí obsahovat: třídu (otaz), třídu s tvary (otaz:kdo|koho), literál (:v|ve|na), vyloučení (l_tvar!spona — „byl“ je hypotézově obojí a identitní otázky patří jinému mechanismu), span uzel+ (1–n tokenů, které graf potvrdí jako entitu; hladově nejdelší, umí ustoupit, nesmí začínat ani končit funkčním slovem), zbytek * a volitelnost ?. Match je ukotvený na celou větu — vzor si nemůže „ukrást“ podobnou větu jiného smyslu. Vykonavatel: jellyai/lang/matcher.py:75.

Pro čitelnost a DRY se vzory píší grok-zkratkami — tabulka pattern_aliases v cs.json:154 (%{TAZACI} → otaz, %{SLOVESO_MINULE} → l_tvar!spona, %{ENTITA} → uzel+…). Zkratky se rozvíjejí rekurzivně (expand_pattern, matcher.py:26); neznámé jméno spadne nahlas — tichá chyba v gramatice je nepřípustná. Pozor (past 14): víceprvková zkratka se vkládá (splice) a posouvá indexy $N v akci.

14.3 Rodiny karet s dimenzemi

Problém, který rodiny řeší: dotazové karty se klonovaly podél skrytých os — q-otaz-minuly / -prezens / -minuly-prodrop… a kombinace, kterou nikdo nenapsal („Kde bydlí?“ = prézens-prodrop), prostě chyběla. Rodinná karta deklaruje kostru + dimenze (enumerativní osy) a deck ji při načtení deterministicky rozvine. Skutečná karta:

// jellyai/iris/patterns/cs/q-otaz.json
{
  "name": "q-otaz",
  "trigger": {
    "event": "utterance.query",
    "pattern": ["%{TAZACI}", "?%{SE}", "<sloveso>", "<ucastnik>"],
    "dimensions": [
      {"slot": "<sloveso>", "values": [
        {"suffix": "-minuly",  "element": "%{SLOVESO_MINULE}"},
        {"suffix": "-prezens", "element": "%{SLOVESO}"}]},
      {"slot": "<ucastnik>", "values": [
        {"suffix": "",         "element": "%{ENTITA}", "priority": 4},
        {"suffix": "-prodrop", "element": null,        "priority": 3}]}
    ]
  },
  "action": {"query": {"hole": "$1", "predicate": "$3", "known": ["$4"]}}
}

Rozvinutí (kartézský součin hodnot os) vyrobí čtyři karty:

flowchart LR
    R["rodina q-otaz<br/>kostra: TAZACI ?SE <sloveso> <ucastnik><br/>dimenze: čas × elipse"]
    R --> A["q-otaz-minuly<br/>„Kdo napsal R.U.R.?“"]
    R --> B["q-otaz-prezens<br/>„Kde bydlí Marcela?“"]
    R --> C["q-otaz-minuly-prodrop<br/>„Kdy se narodil?“"]
    R --> D["q-otaz-prezens-prodrop<br/>„Kde bydlí?“ ← kombinace,<br/>kterou nikdo ručně nenapsal"]

Mechanismus rozvinutí: _expand_family (jellyai/iris/patterns.py:23) — hodnota osy jen dosadí prvek do slotu kostry, přidá příponu jména a smí přepsat prioritu. Prvek null = slot zmizí (pro-drop) — smí být jen posledním prvkem vzoru, aby se neposouvaly odkazy $N; odkaz na zmizelý slot se z known vypustí a v hole/predicate spadne nahlas:

# jellyai/iris/patterns.py:37–55 (zkráceno)
for combo in itertools.product(*(d["values"] for d in dimensions)):
    chosen = {d["slot"]: v for d, v in zip(dimensions, combo)}
    pattern, dropped = [], None
    for index, element in enumerate(skeleton, start=1):
        value = chosen.get(element)
        if value is None:
            pattern.append(element)          # obyčejný prvek kostry
        elif value["element"] is None:
            if index != len(skeleton):
                raise ValueError("prázdný slot musí být posledním prvkem")
            dropped = f"${index}"
        else:
            pattern.append(value["element"])

Zákaz ATN platí i tady: dimenze jsou enumerativní osy, žádné podmínky — hodnota osy nesmí přepisovat akci ani větvit. Empirický korekční nález E1: q-vyberova-prodrop do rodiny nepatří (implicitní spona „být“ je jiná sémantika, ne klon) a svinutí „dekoračních karet“ (q-rekl-adresatovi…) se odložilo, protože osa s přepisem akce by byla program v datech.

Účet E1: dotazových souborů 19 → 16, rozvinutých karet 20, nová kombinace q-otaz-prezens-prodrop zavřela gap („Kde bydlí?“ → Petrovice), teach se píše jednou per rodina.

14.4 Výběr karty: deck.best

V obou světech (dialogové akty i gramatika) vybírá týž mechanismus (patterns.py:198): benefit = těsnost triggeru (kolik podmínek sedí) → priorita → jméno. Dvě jemnosti:

  • Vzorová karta se přes rysovou cestu nikdy nehlásí (patterns.py:160–164) — bez vazby na match by vyhrála „naprázdno“ (vakuová logika, past 2: vsuvka s prioritou 9 kdysi přebila kratke-sloveso 8 a rozbila zápis).
  • U dotazových karet dělá ekvivalentní výběr _card_query (query.py:182) klíčem (priorita, délka vzoru) — a týž klíč používá osvětlení otázkového grafu. Jeden klíč, tři místa — proto se nesmí rozejít (hlídá parity gate).

14.5 Kompilace: karta jako zdrojový kód grafu

Otázkový graf (kapitola 7) je zkompilovaná podoba decku: dotazové karty → uzly otazka, zpřesňovací karty → uzly clarify + hrany, worker karty dodávají texty expertům. Karty se editují, graf se přegenerovává — nikdy naopak. Deck navíc měří kartám telemetrii (použití a měřený zisk aktivace) — z ní by šly váhy uzlů, ale měření ukázalo 0 remíz, takže váhy jsou zatím jen diagnostika.

14.6 Od karty k dotazu: překlad akce

U dotazové karty se vítězné vazby vzoru přeloží na pseudo-QL Pattern (_card_query, query.py:182–297): díru určí tázací token tabulkou interrogatives; predikát projde touž normalizací jako zápis (prefix match proti slovníku grafu — a neznámé sloveso znamená, že se karta nehlásí, past 11: překlep „sworil“ musí propadnout záchranné šabloně); známí účastníci dostanou role (explicitní pár na kartě má přednost, jinak: díra subj → známý je obj, jinak subj); user_subject doplní identitu uživatele; rod slovesného tvaru filtruje pro-drop kandidáty.

14.7 Dekorace vs. karty

Dekorace (chronos:interval, topos:oblast, mnemos:prvni-osoba, role:adresat — kapitola 7.4) nejsou karty: jsou to nároky na části věty, čtené z jazykových tabulek. Hranice je návrhově důležitá: co soutěží o tah, je karta (uzel); co tah omezuje, je dekorace. Trojice karet, které dnes dekorace „zapékají“ jako celé karty (q-rekl-adresatovi, q-kdo-sloveso-misto, q-prvni-osoba-minuly), je známý kompromis — svinutí do dimenzí by vyžadovalo přepis akce osou, což je zakázané (viz §14.3).
