5 · Interní datový model

Struktury, kterými data protékají: od tokenu přes vzor a dotaz k faktu, uzlu a stavu dialogu. Všechny jsou malé, ploché a záměrně „hloupé“ — inteligence je v tom, jak se skládají.

5.2 Jazyková vrstva: TaggedToken

Intuice: slovo, které o sobě poctivě přiznává všechno, čím by mohlo být. Formálně (jellyai/lang/lexer.py:41):

@dataclass(frozen=True)
class TaggedToken:
    form: str                # povrchový tvar („Marcela“)
    norm: str                # deakcentovaný lowercase — klíč do tabulek („marcela“)
    classes: frozenset       # hypotézy druhů: {"jmeno", "l_tvar"}
    l_stem: str = None       # kmen l-ového příčestí („měla“ → „měl“), jinak None

frozen=True není náhoda: tokeny se sdílejí mezi vzory a nesmí je nikdo mutovat. Dvojznačnost se nese dál — rozhodne ji až pozice ve vzoru.

5.3 Karty: PatternCard a PatternDeck

Karta je JSON soubor načtený do PatternCard (jellyai/iris/patterns.py:80): name, trigger (událost + podmínky + případně vzor), dialog (šablona věty), action (co udělat), teach (výklad). PatternDeck je drží seřazené a vybírá vítěze. Detailně kapitola 14.

5.4 Dotazová vrstva: Pattern, SubQuery, Query

Intuice: otázka je neúplný fakt. „Kdo napsal R.U.R.?“ = fakt napsat(subj: ?, obj: R.U.R.), kde ? je díra — role, na kterou se ptáme. Formálně (jellyai/answerer/pattern.py:47):

@dataclass
class Pattern:
    """Neúplný fakt z otázky: predikát, známé role→(termín|SubQuery) a díra."""
    predicate: str = None
    known: list = field(default_factory=list)   # [(role, termín | SubQuery)]
    hole_role: str = None    # subj / obj / loc / time / num / attr / pred
    hole_type: str = None    # person / geo / time / number …

SubQuery umožňuje jedinou rekurzi dotazu: „bratr autora, který napsal R.U.R.“ = najdi autora (vnořený dotaz), pak jeho bratra. Query (jellyai/answerer/query.py:38) balí Pattern se signály pro answerer:

@dataclass
class Query:
    pattern: Pattern = None
    qtype: str = None        # „Kdo“/„Kde“/„Kdy“… (třída odpovědi)
    verb_lemma: str = None
    is_copula: bool = False
    topic_terms: list = field(default_factory=list)
    gender: str = None       # rod slovesného tvaru — filtr pro-drop kandidátů
    place: str = None        # oblast otázky („v Čechách“) — filtr Topos
    card: str = None         # jméno vítězné vzorové karty (telemetrie #38)
    novelty: bool = False    # „Kdo DALŠÍ…?“ — vyluč už jmenované (#53a)

5.5 Faktová vrstva: Fact, Participant, FactNode, Node

Extrakce vyrábí Fact — hodnotový objekt (frozen), jehož identita je (predikát, seřazení účastníci). V grafu se z něj stane FactNode — uzel s vahou (kolikrát se opakoval) a proveniencí. Účastník nese roli a typ (jellyai/graph/extract.py:23, jellyai/graph/graph.py:58–86):

@dataclass(frozen=True)
class Participant:
    role: str    # subj / obj / time / loc / num / pred / attr / theme
    node: str    # id uzlu účastníka
    type: str    # person / geo / time / number / concept / institution

@dataclass
class FactNode:
    id: tuple            # (predicate, participants) — identita faktu
    predicate: str
    weight: int          # kolikrát se fakt v korpusu opakoval
    participants: tuple
    source: set          # zdrojové dokumenty (provenience)

@dataclass
class Node:
    id: str      # kanonické id (entita nebo nominativní lemma)
    type: str    # person/geo/time/number/concept/institution
    weight: int  # v kolika faktech-výskytech figuruje

Ukázka — výrok „Marcela bydlí v Petrovicích“, vysloveno v dialogu 19. 7. 2026:

bydlí( obj:   Marcela   ⟨concept⟩,
       loc:   Petrovice ⟨geo⟩,
       theme: uživatel  ⟨person⟩,     ← pozorovatel, ne účastník!
       time:  19. července 2026 19:53 )

Role theme u dějů značí pozorovatele — metadata výpovědi. Z toho plyne odpovědní zásada: pozorovatel není odpověď („Kdo bydlí v Petrovicích?“ nesmí vrátit „uživatel“). Podrobně kapitola 6.

5.6 Otázkový graf: QNode, QEdge, QGraph, ExpertClaim

(jellyai/iris/qgraph.py:30–52, jellyai/iris/claims.py:14)

@dataclass
class QEdge:
    kind: str        # "zpresneni" | "navrat"
    target: str      # jméno uzlu; "*" = přehraj rozpracovanou otázku

@dataclass
class QNode:
    name: str
    kind: str        # "otazka" | "worker" | "clarify"
    worker: str      # "graf" | "metron" | "chronos" | "iris" | "dialog"
    pattern: list = None           # vzor zdrojové karty (jen kind=otazka)
    card: str = None
    priority: int = 0              # priorita zdrojové karty (klíč decku)
    weight: float = 0.0            # zásahy z telemetrie (#38)
    edges: list = field(default_factory=list)

@dataclass
class QGraph:
    nodes: dict
    predicates: frozenset          # schéma datového grafu (líné instance)
    claims: tuple = ()             # nároky přímých expertů (#57 E2)

@dataclass(frozen=True)
class ExpertClaim:
    name: str        # jméno worker uzlu („metron-vypocet“)
    worker: str      # subsystém, který tah odbaví
    priority: int    # přednost mezi přímými experty
    recognize: object  # callable(text, now) -> bool

Významy v kapitolách 7 a 8.

5.7 Aktivace a stav dialogu

ActivationField (jellyai/graph/activation.py:11) je slábnoucí mapa klíč → jas: warm() přičte, step() vše pohasí (×0.55), hottest() vrátí nejteplejší klíč. Jeden primitiv, tři použití: aktuální subjekt při extrakci (pro-drop), retrieval podle vzdálenosti a konverzační těžiště.

FocusState (jellyai/iris/state.py:46) drží rozpracovaný dialog mezi tahy: PendingFocus (čekáme na volbu kandidáta), PendingIdentity (čekáme na určení podmětu výroku, #43), rozpracovanou připomínku a historii tahů. V modelu otázkového grafu tyto objekty splývají do jediné reprezentace — pozice v grafu (DialogPosition, kapitola 7.6).

5.8 Jazyková data: cs.json

Poslední „struktura“ není Python, ale JSON: jellyai/lang/cs.json nese všechna česká jazyková data — 45+ tabulek. Ukázky (skutečné výřezy):

// cs.json:41 — tázací slovo → (role díry, typ díry, třída odpovědi)
"interrogatives": {
  "kdo": ["subj", "person", "Kdo"],  "koho": ["obj", "person", "Kdo"],
  "kde": ["loc", "geo", "Kde"],      "kdy": ["time", "time", "Kdy"],
  "kolik": ["num", "number", "Kolik"], "jaky": ["attr", null, "Jaký"], …
}

// cs.json:94 — jmenovky rolí pro odpovědní šablony
"role_labels": {"subj": "kdo", "obj": "co", "loc": "kde",
                "time": "kdy", "num": "kolik"},
"empty_role_answer": "O ději „{predicate}“ vím {known}; {missing} nevím.",

// cs.json:154 — grok-zkratky vzorů (DRY + čitelnost)
"pattern_aliases": {
  "TAZACI": "otaz",
  "SLOVESO_MINULE": "l_tvar!spona",
  "ENTITA": "uzel+",
  "PREDL_MISTA": ":v|ve|na", …
}

Zákon 3 — jazyk jako data: nový jazyk = nový JSON (en.json…) + nový adresář karet. Do kódu nikdy nepatří český řetězec pravidla.
