10 · Predikátová schémata a role

Jak systém ví, na co se u kterého děje dá ptát — aniž by kdokoli sepisoval sémantiku ručně. Schéma se čte živě z faktů; otázky jsou jeho stín.

10.1 Role a díra — základ

Připomeňme z kapitol 5 a 6: každý fakt má účastníky v rolích (subj, obj, loc, time, num, pred, attr, theme) a otázka je neúplný fakt s dírou — rolí, na kterou se ptáme. Kterou roli díra nese, určuje tázací slovo tabulkou interrogatives (cs.json:41): „kdo“ → (subj, person), „kde“ → (loc, geo), „kdy“ → (time, time), „kolik“ → (num, number)…

10.2 predicate_roles(): schéma živě z faktů

Intuice: nikdo v projektu nenapsal „predikát napsat má autora a dílo“. Systém to ví z vlastních faktů — stačí se podívat, v jakých rolích účastníci faktů predikátu reálně vystupují. Formálně (jellyai/graph/graph.py:130):

def predicate_roles(self, predicate):
    """Role, ve kterých fakty predikátu nesou účastníky (#57 E3).

    Schéma otázkového grafu se NEkurátoruje — čte se z faktů;
    prázdná množina = graf predikát nezná (žádný verdikt).
    """
    roles = set()
    for fact in self.facts.values():
        if fact.predicate == predicate:
            roles.update(p.role for p in fact.participants)
    return frozenset(roles)

Dvě vlastnosti stojí za podtržení:

  • Živé čtení. Schéma se nečte z kompilátu, ale z aktuálních faktů — predikát, který Mnemos naučil před minutou („Roník žere granule“), má schéma okamžitě. Slovník roste s pamětí.
  • Odvození, ne kurátorování. To je princip „otázky jsou stín schématu faktů“: podpis napsat{subj: person, obj: dílo} implikuje otázky „Kdo napsal X?“, „Co napsal Y?“, „Napsal X Y?“ — sémantika otázkového grafu se neudržuje ručně, protože už existuje ve faktovém grafu.

10.3 Instance: typ otázky × predikát

Uzly otázkového grafu jsou generické typy („kdo-udělal-co“). Teprve spolu s konkrétním predikátem vzniká instance („kdo-napsal-co“) — a ta se nematerializuje předem (2 722 predikátů × typy otázek = tisíce mrtvých uzlů), ale líně: verdikt o smysluplnosti kombinace se spočítá až v okamžiku otázky.

10.4 instance_lit(): verdikt s vakuovým guardem

(jellyai/iris/qgraph.py:136)

def instance_lit(predicate, hole_role, roles_of):
    """Verdikt líné instance (rodina × predikát) nad schématem (E3).

    Returns:
        bool | None: True = role díry ve schématu je; False = fakty
        predikátu roli nikdy nenesou (hledání je marné — chytrá
        clarifikace); None = predikát neznám / bez rolí (vakuový
        guard, past 2 — nesoudit).
    """
    roles = roles_of(predicate) if predicate else frozenset()
    if not roles:
        return None
    return hole_role in roles if hole_role else None

Tři hodnoty, tři významy:

VerdiktKdyDůsledek
Truerole díry ve schématu jehledej normálně (fakt může existovat)
Falsepredikát graf zná, ale roli nikdy nenesehledání je marné → chytrá clarifikace (§10.5)
Nonepredikát neznám / bez rolí / bez dírynesoudit — vakuový guard

Proč vakuový guard (past 2): all(…) nad prázdnou množinou je True — projekt na to draze doplatil, když bezkmenné id „Le“ vakuově „sedělo“ na všechno a pohltilo Ježíše i Marii. U množinových podmínek se vždy ověřuje neprázdnost; instance_lit proto u neznámého predikátu vrací None („nevím“), nikdy False („vím, že ne“).

10.5 Chytrá clarifikace: gap jako vysvětlení

Co se stane, když je verdikt False? Systém neodmítne ani nevrátí generické „nenašel jsem“ — vysvětlí, co o ději ví a co ne. Bydlí to záměrně v answereru, ne v automatu (graph_answerer.py:1109):

def _empty_role_answer(self, pat):
    """Chytrá clarifikace prázdné díry (#57 E3, šablona empty_role_answer).

    Fakty predikátu roli díry NIKDY nenesou → hledání je marné;
    místo generického terminálu vyjmenovat role, které děj zná.
    Vakuové guardy (past 2): bez predikátu, díry nebo rolí se nesoudí.
    """
    from jellyai.iris.qgraph import instance_lit
    predicate = getattr(pat, "predicate", None)
    hole_role = getattr(pat, "hole_role", None)
    if instance_lit(predicate, hole_role,
                    self.graph.predicate_roles) is not False:
        return None
    lang = current()
    labels = lang.get("role_labels", {})
    roles = self.graph.predicate_roles(predicate)
    known = ", ".join(sorted(labels[r] for r in roles if r in labels))
    missing = labels.get(hole_role)
    template = lang.get("empty_role_answer")
    ...
    return Answer(text=template.format(predicate=predicate, known=known,
                                       missing=missing), ...)

Text je šablona v jazykových datech (cs.json:96): "O ději „{predicate}“ vím {known}; {missing} nevím." — jazyk jako data, zákon 3. Výsledek naživo:

❓ Kde napsal R.U.R.?

💬 O ději „napsat“ vím co, kdo; kde nevím.

To je přesně případ, kde otázkový graf poprvé přinesl nové chování, nejen strukturu: před experimentem E3 by otázka skončila generickým terminálem. Oba gap řádky (etalonový i dialogový) tímto zezelenaly.

Proč v answereru, a ne kartou v automatu? Empirický nález z vývoje: první realizace jako dialogová karta prošla jen dialog benchmarkem — etalon měří čistý answerer. Verdikt prázdné díry je odpovědní pravidlo (plyne ze schématu dat), ne dialogový akt — proto patří do answereru, kde ho vidí obě měření. A druhý nález: automat čte prázdnou stopu jako „miss“ a přebíjel verdikt kartou assurance-fail — proto existuje příznak last_empty_role, který říká „tohle je jistota, ne nejistota“ (kapitola 12).

10.7 Role v matchi: preference, filtry a guardy

Jakmile díra roli má, match nad fakty (_match, graph_answerer.py:554) s rolemi zachází jemněji než binárně:

  • Role/typ díry je preference, ne tvrdý filtr — shoda role dává +1000, shoda typu +100 ke skóre; ale sémantické díry (loc, time, num, pred) bez jakékoli shody mlčí („Kdy zemřel?“ nesmí vrátit téma faktu).
  • Časová kotva odpovídá jen časové díře — fakt zapsaný v čase T nesmí nabídnout T jako odpověď na „kdo/co“.
  • Pozorovatel není odpověď — uživatel v roli theme je metadata zápisu.
  • Rolově vázaný známý (#55): dativní adresát otázky („Co řekl Ježíšovi?“) musí být ve faktu v roli theme — dativ nesmí matchnout mluvčího (_theme_bound, graph_answerer.py:572–577).
  • Vztahová jména nejsou identita: sponové „byl bratrem“ je vztah k protistraně — u identitní díry (pred/attr) se vztažná jména bez protistrany přeskakují.

10.8 Souvislost s teorií

Pro čtenáře s lingvistickým zázemím: role účastníků jsou zjednodušená valenční teorie (subkategorizační rámce sloves) — s tím rozdílem, že rámce se neudržují ve slovníku (jako ve FrameNetu či VALLEXu), ale odvozují z korpusu vlastních faktů. Tri-state verdikt instance_lit je pak malá presupoziční kontrola: otázka „Kde napsal X?“ presuponuje, že děj „napsat“ má místo — a systém umí presupozici zamítnout s vysvětlením, místo aby mlčel.
