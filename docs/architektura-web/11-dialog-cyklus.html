11 · Dialog jako grafový cyklus

Dialog není lineární automat („stav 3 → stav 4“), ale cyklus přes tři grafy: každý tah končí tam, kde další začíná — a hrany ven z odpovědi jsou nabídky dalších tahů.

11.1 Cyklus jednou větou

vstup → větný graf → osvětlení otázkového grafu → dispatch (vítěz + dekorace) → dotaz do faktového grafu → odpověď → neosvětlené role odpovědního faktu → nové hrany (nabídky, zpřesnění) → další tah

flowchart TB
    IN["vstup uživatele"] --> VG["větný graf tahu<br/>(tokeny + hypotézy + nároky)"]
    VG --> IL["osvětlení otázkového grafu"]
    IL --> DIS["dispatch: vítězný uzel<br/>+ dekorace (filtry)"]
    DIS --> FG["dotaz do faktového grafu<br/>(match → díra)"]
    FG --> ANS["odpověď"]
    ANS --> ROLES["neosvětlené role<br/>odpovědního faktu"]
    ROLES --> EDGES["hrany ven:<br/>nabídka „Mohu doplnit: kdy“<br/>zpřesnění při přetečení<br/>drill na roli"]
    EDGES --> IN
    ANS -- "aktivace: odpověď + entity svítí" --> FG

Proč to není lineární automat: klasický dialogový automat má stavy definované předem a přechody vyjmenované. Tady je stav pozice v otázkovém grafu + jas faktového grafu — a přechody se z velké části odvozují z dat (role žhavého faktu, oblasti přetečení). Tentýž vstup („Kdy?“) znamená v různých pozicích různé věci: drill po odpovědi, holou otázku bez kontextu, doplnění času připomínky.

11.2 Anatomie jednoho tahu

Skutečné pořadí kroků v _turn() (jellyai/iris/automaton.py:199–360):

  1. Přímí experti — smyčka nad claimy (kapitola 8).
  2. Rozpracované volby — čeká-li dialog na volbu identity podmětu (PendingIdentity) nebo kandidáta (PendingFocus), vstup se čte jako volba; zvolený uzel se zahřeje (+2.0) a původní otázka se přehraje zaostřená.
  3. Věty bez otazníku — příkazová a zápisová větev: připomínky, zapomínání, „zapamatuj si“, správa plánu, posílání, focus-shift, a nakonec konstatování → Mnemos (parse_clauses → karty → fakt(y) → graf + deník).
  4. Vzpomínání a plán — „Co jsem ti řekl včera?“, „Mám nějaké úkoly?“.
  5. Zahřátí intervalu — časové primitivum otázky rozsvítí časové uzly ležící v intervalu ještě před odpovědí.
  6. Odpověď answereru — answer(): rozbor kartou → match → díra; jas před tahem se zapamatuje (jistota se nesmí měřit z jasu, který si tah sám rozsvítil).
  7. Dialogové karty — automat ohlásí událost tahu a karty rozhodnou: resolve.ambiguous (odpověď na hádané volbě → nabídka), data.overflow (přetečení → nabídka oblastí), focus.low (bez odpovědi → upřímný terminál); u čisté odpovědi se případně přilepí proaktivní nabídka rolí (kapitola 13).

11.4 Digging: zpřesnění a návrat jako hrany

Průchod overflow dialogem z pohledu grafu (skutečný scénář „Co řekl Ježíš?“, kapitola 7.5C):

stateDiagram-v2
    [*] --> Qnode : osvětlení („Co řekl Ježíš?“)
    Qnode : uzel otázky q-co-vime
    Clarify : clarify uzel focus-offer-overflow
    Qnode --> Clarify : hrana zpresneni (odpověď PŘETEKLA - 143 výroků)
    Clarify --> Qnode : hrana navrat („Petr“ = volba, otázka se přehraje s pick_focus)
    Qnode --> [*] : odpověď („satane, nemohli jedinou hodinu bdít se mnou“)

Volba oblasti při návratu dominuje: fakt s vybraným uzlem dostává v matchi tier +10000 (pick_focus, graph_answerer.py:618–623) — aktivace sama by remízy jen řadila a to nestačilo („Amen“ přebíjel zvoleného celníka). Volba platí jeden tah.

Navazující holé „Kdy?“ po odpovědi je drill: hrana z odpovědního faktu na jeho dosud neosvětlenou roli. Drill hrany se neenumerují — počítají se z rolí žhavého faktu za běhu (_prev_trace). Fakt narodit(KČ, Hronov, 1890) po otázce „Kde?“ nabízí hrany na time a subj.

11.6 Proč cyklus, ne strom

Za návrhem je jednoduchá observace o reálných konverzacích: uživatel nezačíná každou otázku od nuly — vrtá. Systém proto po každé odpovědi počítá, kudy se dá vrtat dál (neosvětlené role, přetečené oblasti, homonymní kandidáti), a buď je nabídne (proaktivní nabídka), nebo na ně čeká (drill). Graf je pro tohle přirozený formalismus: odpověď není list stromu, ale uzel s hranami ven. A protože hrany se odvozují z dat (rolí faktů), roste dialogový prostor automaticky s daty — nový predikát s časem a místem umí drill „Kdy?“/„Kde?“ bez jediného řádku kódu.
