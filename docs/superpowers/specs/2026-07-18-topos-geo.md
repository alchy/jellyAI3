# Topos-geo — polohové připomínky a mobilní kanál (návrh)

> Zadání (2026-07-18): „až budu v Kauflandu" — geolokace uživatele budí
> reminder Toposu; lokaci místa Topos získá, NEBO si ji checkne podle
> polohy, kde uživatel je. + otázka mobilního frontendu.

## 1. Princip: místo je podmínka jako čas

Připomínka dnes nese termín (`due`). Topos přidá druhou osu — PODMÍNKU
MÍSTA: záznam `{"place": "Kaufland", "task": "koupit prací prášek"}`
se odpálí VSTUPEM do geofence místo tikem hodin. Symetrie subsystémů
platí: Chronos hlídá interval na ose času, Topos kontejnment na ose
prostoru; oba jen odpalují týž sklad připomínek (karty reminder-due,
kanály beze změny).

## 2. Tok polohy

```
mobil (Geolocation API / apka) ──POST /geo {"lat","lon"}──► iris služba
        └► Topos: poloha × gazetteer (místo, souřadnice, rádius)
             ├─ shoda → vstup do místa → odpal polohových připomínek
             └─ neznámé místo + čekající „až budu v X" → DIALOG:
                „Jsi teď v Kauflandu?" → ano → Topos si SOUŘADNICE ULOŽÍ
```

Lokace místa (tři cesty, v pořadí):
1. **Učením za pochodu** (dialog výše) — uživatelova poloha v okamžiku
   potvrzení = souřadnice místa (zadavatelovo „checkne podle lokace, kde
   user je"). Zapisuje Mnemos → záznamy Toposu (`sub_topos_gazetteer`).
2. **Explicitně**: „Kaufland je tady." → aktuální poloha.
3. **Geokódovací adaptér** (volitelný, registr jako kanály): OSM/Nominatim
   dotaz — Topos si lokaci „získá"; adaptér je zásuvný, jádro bez sítě.

Odpal je UDÁLOSTNÍ (při příchodu polohy), ne polling; debounce (např.
2 vzorky v rádiusu) proti GPS šumu; hystereze výstupu z geofence.

## 3. Mobilní frontend — srovnání a doporučení

| Cesta | Geo | Notifikace | Dialog | Náročnost |
|---|---|---|---|---|
| **PWA nad stávajícím webem** ✅ | ✓ Geolocation API | ✓ Web Push | ✓ konzole | malá — web už existuje |
| WhatsApp kanál | ✗ (bez polohy) | ✓ | ✓ obousměrný | střední (Business API) |
| nativní apka | ✓ | ✓ | ✓ | velká |
| „nastěhovat se do mobilu" (celý jelly) | ✓ | ✓ | ✓ | velká (Python v mobilu — Termux) |

**Doporučení: mobil = senzor + displej, mozek zůstává na serveru.**
1. fáze: **PWA** — zeštíhlená stránka (konzole + Reminder okna) nad
   stávajícím viewBase webem, `navigator.geolocation.watchPosition` →
   POST /geo, Web Push pro notifikace; přístup přes LAN/VPN/tunel.
2. fáze: **WhatsApp jako kanál** v existujícím registru kanálů (odchozí
   notifikace + příchozí zprávy do /query) — bez geo, ale všudypřítomný.
3. Nativní apka až kdyby PWA narazila (background geo na iOS).

## 4. Dotčené komponenty (až se bude stavět)

- `services/iris_service.py`: `POST /geo` (ingest polohy) — předá Toposu.
- `jellyai/iris/subsystems/topos.py`: gazetteer záznamy (jméno,
  souřadnice, rádius), kontejnment, geofence match, učení místa dialogem.
- Sklad připomínek: volitelné pole `place` vedle `due`/`event`; výpis
  plánu ukazuje obě podmínky.
- Karty: `reminder-place-set` („⏰ Až budeš v Kauflandu: …"),
  `topos-learn-place` („Jsi teď v X?"), fráze `az budu v`, `az prijdu do`
  v jazykové tabulce.
- Web/PWA: mobilní layout + service worker (Web Push).
