Fáze ② · Dataloader

Z anotace indexy a nahrávání do grafu

Ze shardů vrstvy ① staví pro každý soubor tf·idf index (četnost × unikátnost termínů) a globální idf. Za běhu podle slov rozhovoru vybere nejbližší soubory a nahraje jen je do grafu — ne celý korpus naráz.

Soubor: dataloader.py · třída Dataloader · konfigurace: blok dataloader v config.json · výstup: data/index/<doc>.pkl + _idf.pkl. Koncept viz Dynamické nahrávání.

Příklad: vstup → akce → výstup

vstup — shardy ① 
data/corpus/
  bible_exodus.pkl
  wiki_karel_čapek.pkl
  … (13 souborů)
akce — indexy + výběr 
dl = Dataloader()
dl.build_indexes()
dl.select_files(
  ["faraón","Hospodin"])
výstup — žebříček souborů 
[("bible_exodus", 0.61),
 ("bible_zalmy",  0.28),
 ("bible_genesis",0.13)]
+ data/index/<doc>.pkl

Distinktivní slova (faraón, Hospodin) nasměrují správné soubory; obecná slova skoro nic neváží. Index každého souboru je L2-normovaný, aby délka souboru výběr nezkreslila.

Třída Dataloader

Staví a čte fragmentové indexy a nahrává soubory (shardy) do grafu. Indexy jsou per-soubor na disku a čtou se líně, takže mechanismus unese tisíce souborů: studený soubor = jen index na disku, horký se namountuje.

Dataloader(config_path="config.json")

Načte blok dataloader z JSON configu (cesty ke shardům a indexům, prahy). Uloží si corpus_dir a index_dir pro všechny další metody.

dl = Dataloader()
vstupcesta ke config.json (volitelná) výstupinstance vrstvy ②

build_indexes() → dict

Přepočítá VŠECHNY staty pro dynamické nahrávání z aktuálních shardů: per-soubor tf·idf indexy + globální idf. Je idempotentní — voláš ji po každém přidání raw dat (idf závisí na počtu souborů, proto se přepočítává vše, ne jen nový soubor).

dl.build_indexes()
# [i] 2207260654 : indexy: 13 souborů, 12877 termů, práh výskytů 2
# → {"documents": 13, "terms": 12877}
vstupshardy v corpus_dir výstupindexy v index_dir + počty

select_files(query_lemmas, top_files=None) → list[(doc_id, skóre)]

To je ta brána: seřadí soubory podle shody s dotazem Σ teplo(slovo) × váha_souboru(slovo). Přijme seznam lemmat, nebo dict lemma→teplo; vrací nejteplejší soubory sestupně, jen se skóre > 0, deterministicky (remízy dle jména).

dl.select_files(["spisovatel","drama","hra"], top_files=3)
# → [("wiki_josef_čapek", 0.13), ("wiki_r.u.r.", 0.12), ("wiki_karel_čapek", 0.08)]
vstupslova/teplo dotazu, kolik souborů výstupžebříček (doc_id, skóre)

mount(doc_ids) → dict

Nahraje shardy vybraných souborů a sloučí je do jednoho korpusu pro stavbu grafu (jas + hrany). Unmount = tento slovník nedržet / u trvalého grafu zahodit fakty se source=doc.

corpus = dl.mount(["wiki_r.u.r.", "wiki_karel_čapek"])
# výstup: sloučené shardy — {(doc, idx): {entities, sentences}}
len(corpus)                  # → 373  (59 + 314 vět)
corpus[("wiki_r.u.r.", 1)]   # → {"entities": [], "sentences": [[…]]}
vstupseznam doc_id k nahrání výstup{(doc,idx): rec} přes vybrané soubory

document_ids() → list  ·  load_shard(doc_id) → dict  ·  load_idf() → dict

Pomocné čtení: seznam dostupných souborů (jen názvy, levné i u tisíců), jeden shard z disku, globální idf tabulka. Slouží výběru i mountu a čtou se líně.

dl.document_ids()   # → ["bible_exodus", …, "wiki_válka_s_mloky"]
dl.load_idf()["n"] # → 13 (počet souborů)
vstupdoc_id / nic výstupseznam id / shard / idf tabulka
