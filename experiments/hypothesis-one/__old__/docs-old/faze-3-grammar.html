Ze surové anotace naše role a VZOR

Tady se surové názvosloví UDPipe (deprel = nsubj/obl…) NORMALIZUJE na naše klíče rolového katalogu (who/where/action/…) a staví se VZOR (frame_sig). Pracuje nad už anotovanou větou z korpusu ①.

Soubory: grammar_vzor.py (GrammarVzor — primitiva) · role_catalog.py (RoleCatalog — rozklad na role). Klíče: viz Termíny → Rolový katalog.

Příklad: vstup → akce → výstup

tokeny s deprel/upos/feats

 města Kafarnaum a učil

  .standardize(sent)
# deprel + pád + předložka
# → náš klíč
výstup — sektory s naším klíčem 
[(Odešel,   action),

Plná tabulka rozkladu (# / sektor / UPOS / lemma / pád / náš klíč / co je špatně) je v Termínech → Klíče v použití. Chybu Kafarnaum → who dělal UDPipe 1; nad novým korpusem (UDPipe 2) dává role_catalog správně Kafarnaum → where.

Třída GrammarVzor — primitiva VZORu

Drží parametry VZORu (poloměr okna, interpunkce, modalita) a jazyková data. Metody jsou čisté transformace nad tokeny; VZOR je přesný na gramatiku, abstraktní na lexém.

slot(tok) → str

Match-obálka jednoho tokenu: UPOS + nosné rysy (pád, čas). Pád v pivotu JE role, proto se do slotu vždy bere.

g = GrammarVzor()
g.slot({"upos":"PROPN","feats":{"Case":"Nom"}})   # → "PROPN:Nom"
vstuptoken (dict)výstupSLOT (str)

frame_sig(toks, i, modality, r=None) → str

VZOR — skládá se z r SLOTŮ vlevo · pivot · r SLOTŮ vpravo · modalita. Okraje věty doplní ^ (začátek) / $ (konec). Poloměr r řídí, jak široké okolí pivotu se do vzoru vezme.

g.frame_sig(sent, 2, ".", r=2)
# → "NOUN:Nom·ADP:Loc·NOUN:Loc·VERB:Pres·ADP:Ins·."
vstupvěta, index pivotu, modalita, poloměrvýstupVZOR (str)

Jak se uplatňuje r — a proč r = 2

Vezměme větu „Autor v příběhu varuje před …" a pivot „příběhu" (NOUN:Loc, role where). Se zvětšujícím se r roste okno kolem pivotu:

rokno kolem pivotu „příběhu"VZOR
1v · příběhu · varujeADP:Loc·NOUN:Loc·VERB:Pres·.
2Autor · v · příběhu · varuje · předNOUN:Nom·ADP:Loc·NOUN:Loc·VERB:Pres·ADP:Ins·.
  • r = 1 vidí jen bezprostřední sousedy — předložku v (nese pád Loc) a přísudek. To sice zachytí, že jde o předložkovou frázi, ale ne její místo v klauzuli: týž vzor ADP:Loc·NOUN:Loc·VERB sdílí spousta nesouvisejících vět → match je nejednoznačný.
  • r = 2 přidá VLEVO podmět Autor (NOUN:Nom) a VPRAVO další argument před (ADP:Ins) — tedy celý skelet klauzule (podmět · pivot · přísudek · další argument). To je přesně to, co roli nese.
Proč zrovna r = 2 (měřeno): shadow harness (sim_r_compare) porovnal překryv VZORů mezi otázkou a jejím faktem při r=1 vs r=2. Při r=2 přežije víc přesných vzorů → správný fakt se trefí s vyšší jistotou. r=1 je příliš úzký (vzor sdílí mnoho kontextů → nejednoznačno); r=3+ naopak přefragmentuje — skoro každý vzor je unikát (hapax), mizí generalizace a pokrytí padá. r=2 je rovnováha: dost kontextu na roli, a přesto vzor zobecňuje. Proto rolová determinace (synth_determination) i kurátorská DB stavějí VZOR na r=2.

role_key(deprel, upos, feats, prep="", nominal_pred=False) → str|None

Tady se surový deprel normalizuje na náš klíč (jediný zdroj mapování, data v cs.json). Předložka může roli přebít; vrací None, když deprel nemapujeme.

g.role_key("obl", "NOUN", {"Case":"Loc"})   # → "where"
g.role_key("obj", "NOUN", {"Case":"Acc"})   # → "whom_what"

Třída RoleCatalog — rozklad na role

Rozkládá anotovanou větu na sektory s naším klíčem a na klauzule. Používá primitiva GrammarVzor, takže stejný význam = stejný klíč napříč systémem.

standardize(sent) → list[(token, klíč)]

Rozklad věty na sektory — každý token dostane náš klíč, deprel se ven nedostane. VERB → action, jmenný přísudek se sponou → state, strukturální deprel → strukturální klíč.

RoleCatalog().standardize(sent)
# → [(Odešel, "action"), (do, "preposition"), (Kafarnaum, "where"), …]
vstupanotovaná větavýstup[(token, náš klíč)]

curated_standardize(sent, r=2) → list[(token, klíč, kurátorováno?)]

Jako standardize, ale VZOR okna se hledá v kurátorské DB (curated.json) — je-li tam, bere se OPRAVENÁ role. Tak se řeší systematické chyby anotátoru.

RoleCatalog().curated_standardize(sent)
# → [(Kafarnaum, "where", True), …]   # True = opraveno kurátorem
vstupanotovaná věta, poloměrvýstup[(token, klíč, opraveno?)]

decompose(sent) → list[clause]

Rozloží větu na KLAUZULE (přísudek + jeho role); relativní klauzule dědí místo z antecedentu (meziklauzulová inference #59). Základ pro syntézu faktů (④).

RoleCatalog().decompose(udpipe_sent_z_korpusu)
# → [{"predicate":"odejít", "roles":{"action":["odejít"], "where":["Kafarnaum"]}}, …]
vstupanotovaná větavýstupklauzule {predicate, roles…}

role_of(t, sent, byid) → str|None  ·  standard_role(t, sent, byid, cop_heads=None) → str

Obsahová role jednoho tokenu (souřadný konjunkt dědí roli řídícího) a plný standardizovaný klíč jednoho sektoru. Volají je standardize i decompose.
