Fáze ④ · Synthesis

Grafikon FAKT ↔ OTÁZKA a slovník VZOR → role

Z rozkladu na role (③) se staví syntetické vazby: pro každou větu vazba FAKT↔OTÁZKA (registr) a přes celý korpus začištěný slovník VZOR→role. To jsou data, nad kterými pak běží match a výběr odpovědi (⑤).

Soubory: synth_registry.py (SynthRegistry) · synth_determination.py (SynthDetermination). Vstup: nový korpus přes Dataloader.mount. Výstup: registry.jsonl, determination.json.

Příklad: vstup → akce → výstup

vstup — anotovaný korpus 
data/corpus/*.pkl
věty s rozkladem na role (③)
akce — syntéza 
SynthRegistry().run()
SynthDetermination().run()
výstup — grafikon + slovník 
registry.jsonl
  9885 vazeb FAKT↔OTÁZKA
determination.json
  4804 CONFIRMED VZOR→role

Třída SynthRegistry — vazby FAKT ↔ OTÁZKA

Pro každou větu vyrobí vazbu na fakt: predikát, díra (podmět/účastník), odpovědní sloty (lemma + náš rolový klíč + rod) a kontext. Odpovědi jsou v základním tvaru; rolový klíč je týž jako v ③ (jednotný mapovač).

run() → dict

Projde celý korpus a zapíše registry.jsonl (jedna vazba na řádek). Nejdřív spočte podmíněný fold PROPN, pak z každé věty vytvoří vazbu.

SynthRegistry().run()
# [i] … : vazeb FAKT↔OTÁZKA: 9885, predikátů: 1863, answer-slotů: 26121 (⌀ 2.6/fakt)
# → {"bindings": 9885, "predicates": 1863, "answer_slots": 26121}
vstupkorpus (z Dataloader) výstupregistry.jsonl + souhrn

_binding(sent, doc, si, fold) → dict|None

Z jedné anotované věty vazba, nebo None (žádný přísudek/odpověď). Predikát je root VERB (jinak spona = identitní fakt); sloty jsou argumentová jména se svým rolovým klíčem.

sr._binding(sent, "bible_lukas", 12, fold)
# → {"predicate":"odejít", "hole":"subj/person",
#    "answers":[{"lemma":"Kafarnaum","role":"where","q":"Kde","gender":None,"name":True}],
#    "context":[…], "doc":"bible_lukas", "sent":12, "text":"Odešel do …"}
vstupvěta, doc_id, index, fold PROPN výstupvazba {predicate, answers, context…} / None

_propn_fold(corpus) → dict  ·  _skip(lem, upos) → bool

Podmíněný epentetický fold PROPN (Čapk→Čapek, ale jen když „Čapek" je reálně častější — nekorumpuje Egypt→Egypet) a filtr neslov (interpunkce, jednopísmenné iniciály).

sr._propn_fold(corpus)   # → {"Čapk": "Čapek", …}
vstupkorpus / (lemma, upos) výstupfold dvojice / bool

Třída SynthDetermination — slovník VZOR → role

Agreguje role per VZOR přes celý korpus a povýší jen ZAČIŠTĚNÉ: PURE (jediná role na VZOR) + FREQUENT (≥ prahu) + content-role. Revize je konzistence, ne správnost; curated.json opravuje známé chyby, impure/vzácné zůstávají kandidáty.

run() → dict

Projde korpus a zapíše determination.json. Vrací souhrn (počet VZORů, CONFIRMED, pokrytí content-tokenů).

SynthDetermination().run()
# [i] … : VZORů: 69426, CONFIRMED (pure & ≥3): 4804, pokrytí: 24983/103025 (24%)
vstupkorpusvýstupdetermination.json + souhrn

aggregate(sentences) → {VZOR: Counter(role→počet)}  ·  confirmed(vzor_roles) → {VZOR: role}

Spočte rozdělení rolí na každý VZOR (jen content-role, r=2 — viz Grammar → poloměr r) a vybere PURE + FREQUENT VZORy. Rozdělení kroků umožňuje ladit práh bez opakované agregace.

vr = sd.aggregate(sents)
sd.confirmed(vr)   # → {"ADJ:Gen·NOUN:Gen·…": "where", …}
vstupvěty / agregacevýstupagregace / {VZOR: role}
