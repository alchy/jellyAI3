Použití — ptát se, přidávat data, reindexovat

Praktický návod: jak systém spustit, jak mu poslat dotaz, jak doplnit vlastní text a jak po přidání dat přepočítat indexy. Systém je deterministický Q&A nad českým textem — na známé otázky odpoví, u nejasných se doptá, a když neví, nehádá.

Předpoklad: Python 3 a dostupná služba UDPipe 2 (rozbor věty). Runtime je jinak offline a deterministický — stejná otázka dá vždy stejnou odpověď. Pracovní adresář: experiments/hypothesis-one/.

1. Zeptat se (nejběžnější)

Spusť interaktivní dotaz a piš otázky:

$ python3 ask.py Ptej se (prázdný řádek ukončí). Např.: Kdo napsal Švejka? ? Kdo napsal Švejka? → Hašek ? Kde se narodil Karel Čapek? → Svatoňovice ? Kdo je pes domácí? → šelma ? Kdy měla premiéru hra R.U.R.? nejsem si jist — mám data o: 1921, 1932, pes. Upřesni prosím?

Prázdný řádek (nebo Ctrl-D) ukončí. Dobré otázky začínají tázacím slovem: Kdo, Co, Kde, Kdy, Kolik, Čím… (systém z něj pozná, co hledáš — osobu, věc, místo, čas).

Tři odpovědní režimy

režimkdyvypadá
odpověďjasný vítěz→ Hašek
doptánívíc rovnocenných možnostínejsem si jist — mám data o: X, Y. Upřesni?
nevímslabá / žádná evidenceo tom nemám jasná data (nehádám)

To je záměr: systém raději řekne „nevím" nebo se doptá, než by si vymyslel odpověď.

2. Přidat vlastní data

a) Vlož text

Ulož obyčejný český text jako .txt do složky ../../data/raw/. Název souboru = název dokumentu (bez diakritiky v cestě není nutný, ale drž se malá_pismena.txt).

$ cp muj_clanek.txt ../../data/raw/milan_kundera_zivot.txt

b) Zanotuj (rozbor vět)

Spusť anotaci — projde data/raw/*.txt a z každého udělá jeden anotovaný „shard" (data/corpus/<doc>.pkl). Je přírůstková: už zanotované a nezměněné soubory přeskočí, zpracuje jen nové/upravené. (Vyžaduje UDPipe 2.)

$ python3 annotate_corpus.py [i] … : anotace: dokumentů 54, model(pin) czech-pdtc-ud-2.17-… [i] … : hotovo: … vět, přeskočeno 53 (přírůstkově), nový 1

Pozn.: vstupní data se nikdy nepřepisují; anotace je cache. Zkratky („R.U.R.") se automaticky slučují do jednoho tokenu.

3. Reindexovat (po přidání dat)

Když přibyl nebo se změnil text, přepočítej odvozené staty. Spusť v tomto pořadí (vše lokální, bez sítě):

$ python3 synth_registry.py # ④ fakty (kdo/co/kde/kdy z vět) $ python3 -c "from dataloader import Dataloader; Dataloader().build_indexes()" # ② tf·idf + entita-jako-podmět + doc-graf $ python3 build_facts_all.py # fakty + koreference + datumy (Chronos) $ python3 build_templates_all.py # VZOR šablony otázka↔fakt

Po reindexaci se nový obsah rovnou promítne do odpovědí (python3 ask.py). Indexy závisí na počtu dokumentů (idf, doc-graf), proto se přepočítávají celé — ne jen nový soubor.

Jednou větou: přidat = txt do data/raw/ → annotate_corpus.py; reindexovat = registry → indexy → fakty → šablony; ptát se = ask.py.

Co systém umí (a neumí)

  • Umí: faktické otázky nad textem — kdo něco napsal, kde/kdy se narodil, co něco je, kolik něčeho je. Když si není jist, doptá se; když neví, přizná to.
  • Neumí (zatím): řetězit kontext mezi otázkami („A jeho bratr?"), teologická/nefaktická tvrzení, otázky bez tázacího slova. Otevřené body: Otevřené otázky.
