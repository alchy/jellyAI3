Fáze ⑤ · runtime matcher (answering.py)

⑤ Queryparser — živá otázka → odpověď

Srdce běhu: živá otázka projde jedním tahem dialogového automatu — rozsvítí graf, sesbírá kandidáty (fakty + šablony), assurance rozhodne jasno / nejasno / nehádej, a systém odpoví, doptá se, nebo přizná „nevím". Jediná živá anotace je UDPipe 2 (rozbor otázky); rozhodování je symbolické a deterministické.

Celý tah v kostce: _parse (otázka → tokeny) → _question (query-VZOR + lemmata + role díry) → _answer_role (kopula → komplement) → _predicate (predikát) → select_files+mount (② brána) → feed/spread (světlo) → _candidates (predikát+role ∪ VZOR) → _assurance (jasno/nejasno) → odpověz / doptej / nehádej → reinforce.

Rozbor otázky

_parse(question) → tokeny

Pošle otázku na UDPipe 2 (jediná živá anotace), rozparsuje CoNLL-U a sloučí zkratky stejně jako korpus (chokepoint — jinak by „R.U.R." bylo „R/U/R" a nesedělo na fakty).

vstup„Kdo napsal Švejka?" výstup[Kdo(PRON,Int), napsat(VERB,root), Švejk(PROPN,obj), ?]

_question(toks) → (query-VZOR, lemmata, role díry)

Najde tázací slovo (rys PronType=Int), z jeho okna udělá query-VZOR (stín otázky) a hlavně určí roli díry — přes TÝŽ mapovač RoleCatalog.role_of jako fakty (DRY): „Kdo" = who, „Co napsal" = whom_what, „Kde" = where, „Kdy" = when. Lemmata = obsahová slova bez tázacího.

„Kdo napsal Švejka?"role díry = who, lemmata = [napsat, Švejk]

_answer_role(toks, hole_role) → efektivní role odpovědi

Kopulová identita „Kdo/Co JE X?" se neptá na who-podmět (X je známý), ale na to, co X JE — komplement (role state). Platí i když role_of vrátí None (tázací slovo je jmenný přísudek). Bez tohoto by „Kdo je pes?" vrátilo nejteplejší who-podmět z jakékoli věty „X je Y" (v korpusu s biblí → „bůh").

„Kdo je pes domácí?"role who → state (hledá komplement: šelma)

_predicate(toks) → lemma predikátu

Predikát otázky = lemma hlavního slovesa (root, jinak první VERB), jinak spona. Je to JOIN klíč parent-modelu: „Kde se narodil X" a fakt „Narodil se v Y" se potkají přes narodit, nezávisle na slovosledu.

„Kde se narodil Karel Čapek?"predikát = narodit

Kandidáti (dvě cesty)

_candidates(q_vzor, predicate, hole_role, known) → [kandidát]

Sesbírá kandidáty DVĚMA cestami do jednotného tvaru {answer, role, doc, mq, kind, fact_ref}: pred (parent-model: FactStore.match(predikát, role)) a vzor (window-VZOR: TemplateStore.match_scored). Predikátová cesta je PŘESNÁ: přijme jen fakt, který sám nese ZNÁMOU entitu otázky v JINÉ roli než odpověď — jinak by predikát „napsal"+who zaplavil všemi autory (a glow by mispicknul). Ta entitní vazba v jiné roli je zámek proti slepenci (who=[Švejk, Olbracht] není vazba; who=[Karel] + where=? je).

„Kde se narodil Karel Čapek?"pred: narodit-fakt s who=[Karel] → where=[Svatoňovice, rodina]

_known_overlap(fact_ref, role, known) → počet

Kolik ZNÁMÝCH entit otázky nese fakt kandidáta v jiné roli než odpovědní. Sdíleno oběma cestami (fakt dohledán přes FactStore.by_ref). Co-filler odpovědní role se nepočítá (proti slepenci). Signál base v _rank.

fakt narodit {who:[Karel], where:[Svatoňovice]}, díra where, known={karel,čapek}→ 1 (Karel ve where≠roli)

Rozhodnutí — glow-orders-ties + assurance

_rank(cands, known, hole_role) → nejlepší kandidát

Glow-orders-ties jako parent: STRUKTURA rozhoduje první, aktivace (glow) řadí jen remízu — leaked světlo tak NEMŮŽE přebít strukturní shodu. Vybírá max podle base tuple (sestupně):

#složkaco znamená
1echo_okodpověď NENÍ slovo z otázky (self-answer „Kdo je pes?"→pes je nesmysl)
2role_fitrole kandidáta = role díry (Hašek/who > román/what_subject)
3known_ovfakt nese známou entitu otázky (Švejk → Hašek, ne libovolný autor)
4def_fitDEFINIČNÍ komplement — state ze sent 0 (úvodní věta = definice: „Josef Čapek byl malíř")
5type_fitTYP entity sedí na díru (Topos: where → MÍSTO; Svatoňovice > rodina)
6kind_predparent-model (predikát+role) bije window-VZOR
7mqexaktní shoda VZORu > fuzzy
8glowteplo aktivace (POSLEDNÍ — jen remíza)

Klíč def_fit (dřív chyběl v dokumentaci): mezi mnoha state-komplementy entity vybere ten z ÚVODNÍ věty = definici (jako biografická závorka bere jen sent 0).

_assurance(cands, known, hole_role) → (mode, winner, offer)

Jasno vs nejasno (parentovo assurance). Deduplikuje kandidáty dle odpovědi, seřadí dle (base, glow), a rozhodne podle ODSTUPU vítěze od druhého — STRUKTUROU (base_sep) NEBO SVĚTLEM (glow_sep: vítěz má > 2× glow druhého). Klíč: vítěz s dominantním světlem je jistý i při remíze base (jinak by „Kdo napsal Švejka" zbytečně doptával — Hašek má 20× glow nad Vančurou).

modekdyvýstup
answerstrukturní síla ≥ 1 A odstup (base NEBO glow)odpověz
clarifyevidence, ale rovnocenní rivalové„mám data o X, Y — upřesni?"
unsurejen echo, žádná struktura„nemám jasná data" (nehádej)

Celý tah

answer(question, carry_context=False) → dict | None

Celý runtime tah: rozbor → select_files (② brána dle otázky) → mount horkých (šablony i fakty, #60) → build_graph/feed/spread (světlo) → _candidates → _assurance → jen u answer zpětně přihřeje (reinforce, řetěz). Bez horkých souborů nebo bez záchytu vrací None.

carry_context=True (interaktivní dialog, ask.py/viz.py): světlo i mount PŘETRVAJÍ mezi tahy. Navazující otázka bez vlastní entity zdědí téma z horkých entit minula a jen pohasne (decay); otázka s vlastní entitou téma přepne (žár i mount zhasnou). Mechanismus a proč: aktivace v dialogu §4b. Etalon volá s False (samostatný tah, deterministicky srovnatelný).

vstup„Kde se narodil Karel Čapek?" výstup{answer: „Svatoňovice", mode: „answer", offer: […], fact_ref: [„wiki_karel_čapek", 4], hole_role: „where", candidates: N, via: „pred"}

Podklad: fakt-store (predikát+role)

FactStore.match(predicate, hole_role) → [(lemma, Fact)]

Parent-model JOIN: fakt a otázka se potkají PREDIKÁTEM + rolí díry, nezávisle na slovosledu (window-VZOR to nesouměří — vidí jen lokální okno). Fact = predikát + role-sloty + provenience; shardy per soubor, mount jen horkých (#60), index by_predicate + by_ref (ten krmí _known_overlap). Doc-kontext supluje pro-drop podmět.

Podklad: experti (podgrafy)

Topos.is_place(lemma) — expert MÍSTO

Rozliší, zda lemma je MÍSTO (gazetteer GEO entit z NER) → signál type_fit pro výběr správného where-slotu (Svatoňovice ✓ / rodina ✗). Detail: fúzní studie.

Chronos.temporal_facts(sent, grammar) → [(predikát, rok)] — expert ČAS

Rozpozná ROK (4místné NUM v rozsahu) a přiřadí ho k HLAVNÍMU predikátu věty (root verb) → fakt dostane when-slot i z BĚŽNÉHO textu (ne jen z biografické závorky). „Premiéra … měla proběhnout … došlo k odložení termínu na 25. ledna 1921" → rok visí přes conj pod root měla → mít-fakt dostane when=1921 → „Kdy měla premiéru?" se trefí. (Zatím jen rok; plné datum/interval/relativní čas jsou rozšíření — parent je má.)
