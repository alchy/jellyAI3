Koncept · #60 fragmentový graf

Dynamické nahrávání souborů podle matche

Korpus se nedrží celý v paměti. Každý soubor je samostatná jednotka; za běhu se do grafu nahrají jen ty soubory, které odpovídají rozhovoru — vybrané frekvenční analýzou termínů (četnost × unikátnost, tf·idf). Proto se anotace ukládá po souborech, ne do jednoho picklu.

Proč ne jeden pickle: monolit nutí načíst všechno naráz. Při tisících souborů to nejde do paměti a hlavně to bere smysl cílení — nemůžeš „rozsvítit" jen relevantní soubory. Shard per soubor + index per soubor to řeší.

Formát ukládání

cestacoformát
data/corpus/<doc>.pklanotace jednoho souboru (mountovatelná jednotka)pickle {(doc,idx): {entities, sentences}}
data/index/<doc>.pklpředdefinovaný index souboru: slovo → váha vůči ostatnímpickle {lemma: tf·idf}, L2-normovaný
data/index/_idf.pklglobální tabulka unikátnosti termínůpickle {"idf": {lemma: idf}, "n": počet}

Každý soubor MUSÍ mít svůj index. Bez něj ho brána neumí ohodnotit a nahrát — proto je index součást formátu, ne dodatek. Indexy jsou shardovatelné (jeden soubor = jeden index), takže mechanismus unese tisíce souborů: studený soubor = jen index na disku, horký se namountuje.

Co je četnost × unikátnost (tf·idf)

  • tf (term frequency) = jak často slovo v souboru je — o čem soubor je.
  • idf (inverse document frequency) = jak vzácné slovo je napříč korpusem — log(N/df)+1. Slovo ve všech souborech (df=N) má idf≈1 (nerozlišuje); slovo v jednom souboru (df=1) má idf vysoké (silně distinktivní).
  • tf·idf = váha „tento soubor nese toto slovo silněji než ostatní". Distinktivní slova (faraón, drama) nasměrují správný soubor; obecná (být, člověk) skoro nic neváží.

Loader do grafu — funkce (dataloader.py, třída Dataloader)

build_indexes() → dict

Přepočítá VŠECHNY staty pro dynamické nahrávání z aktuálních shardů: per-soubor tf·idf indexy + globální idf. Je idempotentní — voláš ji po každém přidání raw dat (viz Anotační cache → přidání obsahu).

Dataloader().build_indexes()
# [i] 2207260654 : indexy: 13 souborů, 12877 termů, práh výskytů 2
vstupshardy v corpus_dir výstupindexy v index_dir + počty

select_files(query_lemmas, top_files=None) → list[(doc_id, skóre)]

To je ta brána: seřadí soubory podle shody s dotazem Σ teplo(slovo) × váha_souboru(slovo). Přijme seznam lemmat, nebo dict lemma→teplo (žhavost z těžiště rozhovoru); vrací nejteplejší soubory sestupně, deterministicky.

Dataloader().select_files(["spisovatel","drama","hra"], top_files=3)
# → [("wiki_josef_čapek", 0.13), ("wiki_r.u.r.", 0.12), ("wiki_karel_čapek", 0.08)]
vstupslova/teplo dotazu, kolik souborů výstupžebříček (doc_id, skóre)

mount(doc_ids) → dict

Nahraje shardy vybraných souborů a sloučí je do jednoho korpusu — vstup pro stavbu slovního grafu (jas + hrany). Unmount = tento slovník nedržet (u trvalého grafu zahodit fakty se source=doc).

corpus = Dataloader().mount(["wiki_r.u.r.", "wiki_karel_čapek"])
len(corpus)   # záznamy obou souborů dohromady
vstupseznam doc_id k nahrání výstup{(doc,idx): rec} přes vybrané soubory

load_shard(doc_id) → dict  ·  document_ids() → list  ·  load_idf() → dict

Pomocné čtení: jeden shard z disku, seznam dostupných souborů (jen názvy, levné), globální idf tabulka. Slouží mountu i výběru a čtou se líně.

Dataloader().document_ids()   # → ["bible_exodus", …, "wiki_válka_s_mloky"]
vstupdoc_id / nic výstupshard / seznam id / idf tabulka

Pasti (dle spec #60): zpětný tok jen bonusem při vážení evidence, ne plošným zahříváním slov (runaway/rozmazání domény); fragmentový prior je patro, ne veto — přímá evidence dotazu vždy přebíjí, konflikt → clarifikace; idf nad malým korpusem je vratké → práh výskytů; skóre vždy řadit (determinismus).
